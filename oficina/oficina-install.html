Descaracterizacao de TV Box TX9 Pro

Do Android ao Armbian Linux — passo a passo direto

Receita Federal IFSC Amlogic S905W

Apresentador: Romulo de Aguiar Beninca

O Projeto de Descaracterizacao

  • ▸Origem: TV Boxes apreendidas pela Receita Federal e destinadas ao IFSC para reaproveitamento.
  • ▸Dispositivo: TX9 Pro — SoC Amlogic S905W (ARM Cortex-A53), 1 GB RAM, 8 GB eMMC.
  • ▸Objetivo: substituir o firmware Android inseguro por Armbian Linux, mantendo todo o hardware funcional.
  • ▸Resultado: dispositivo seguro, controlavel e reaproveitado como no Linux, mini-servidor ou laboratorio de containers.
Descaracterizar = remover completamente o sistema original (Android) e instalar um sistema operacional limpo, auditavel e mantido pela comunidade.

Todos esses componentes foram mapeados a partir do DTB extraido do Android original e validados no Armbian.

Por que remover o Android?

Risco no Android originalComo o Armbian resolve
Processos root desconhecidos em backgroundSistema limpo, controle total do que executa
Sistema de arquivos montado em rw (inseguro)Montagem controlada e endurecimento
Versao mascarada: reporta Android 10, e Android 7.1.2 / kernel 3.14Kernel moderno 6.x mantido pela comunidade
Sem atualizacoes de segurancaAtualizacoes via apt
Possiveis backdoors no firmware de fabricaSistema auditavel, repositorios abertos
Por que nao basta instalar um app sobre o Android? Uma camada de compatibilidade nao elimina as vulnerabilidades nem o software malicioso do firmware original. A descaracterizacao precisa ir ate o kernel.
O Android original nao e confiavel. A unica forma segura de reaproveitar o hardware e substituir o sistema inteiro — do bootloader ao userspace.

Por que descaracterizar e dificil?

  • ▸Nao e so "instalar Linux": o hardware do TX9 nao e padronizado como um PC. O kernel precisa de um Device Tree Blob (DTB) especifico para cada placa.
  • ▸Wi-Fi sem suporte mainline: o chip SSV6051 nao tem driver no kernel oficial — precisa ser compilado out-of-tree para cada versao de kernel.
  • ▸Display frontal proprietario: o FD6551 e controlado por bit-bang via GPIO, sem driver Linux existente — foi necessario criar um do zero.
  • ▸DTB inexistente: nenhuma imagem Armbian generica traz o mapeamento correto dos perifericos do TX9 — foi preciso extrair o DTB do Android, descompilar e correlacionar pinos.
O trabalho pesado ja foi feito. A imagem que vamos usar hoje inclui o DTB correto, o driver Wi-Fi compilado, o servidor do display e o daemon IR — tudo pronto para funcionar.

Material necessario: notebook Linux, cartao microSD (min. 8 GB), leitor de cartao USB, TV Box TX9 Pro, cabo de rede.
O que a imagem ja inclui:
  • Armbian + kernel 6.x + DTB correto para TX9 Pro (S905W / P281)
  • Driver Wi-Fi SSV6051 compilado
  • Servico do display frontal (tx9-display.service)
  • Servico do controle remoto IR (tx9-ir.service)
  • Docker instalado e pronto

1Gravar a imagem no cartao SD

1.1 — Baixar a imagem (ja inclui tudo: Armbian + DTB + drivers + servicos + Docker):

https://drive.google.com/file/d/1u4GrDhsN_l2zTHFpy2P6BQBlMFfIhFet/view?usp=sharing

1.2 — Baixar o script de gravacao:

wget https://rbeninca.github.io/tvbox/oficina/gravar-cartao.sh

1.3 — Identificar o cartao e gravar:

# identifique o cartao SD (CUIDADO: confira o dispositivo!)
lsblk

# grave a imagem (substitua /dev/sdX pelo seu cartao)
sudo bash gravar-cartao.sh tx9-pro-armbian-reduzido.img /dev/sdX
O script apaga todo o conteudo do cartao. Confira o dispositivo com lsblk antes. Apos gravar, ele verifica automaticamente a integridade com cmp.
Leitor USB lento? Limite a velocidade: RATE=10M sudo bash gravar-cartao.sh tx9-pro-armbian-reduzido.img /dev/sdX

2Bootar pelo cartao SD

  1. Desligue a TV Box (retire da tomada).
  2. Insira o cartao SD gravado.
  3. Com um palito, pressione e segure o botao escondido no conector AV (parte inferior/traseira).
  4. Sem soltar o botao, ligue a TV Box na tomada.
  5. Segure por 5 segundos e solte.
  6. O Armbian vai iniciar pelo cartao.

2Descobrir o IP — ifrede.apk

Se a TV Box ainda estiver no Android e voce precisar do IP para conectar via rede, instale o app ifrede.apk — ele exibe o endereco de rede no display frontal da caixa.

Opcao A — Instalar via navegador (na TV Box)

https://rbeninca.github.io/tvbox/oficina/ifrede.apk

Abra no navegador Android, baixe e instale. Pode ser necessario habilitar "Fontes desconhecidas".

Opcao B — Instalar via cabo USB (com ADB)

wget https://rbeninca.github.io/tvbox/oficina/ifrede.apk
adb install ifrede.apk
Abra o app ifrede na TV Box. O IP aparecera no display frontal (ex.: 192.168.1.50).

3Instalar na eMMC

Com o Armbian rodando pelo cartao SD, conecte-se e execute o script de instalacao:

# conecte via SSH (ou use o terminal local)
ssh root@IP_DA_TVBOX       # senha: aluno123

# rode o script de instalacao na eMMC
/root/instal-aml.sh

O script copia o sistema do cartao para a memoria interna (eMMC), configura o bootloader e prepara tudo para funcionar sem o cartao.

Operacao destrutiva: apaga o Android da eMMC permanentemente. So execute apos confirmar que o boot pelo cartao esta funcionando.

Apos a instalacao:

  1. Desligue a TV Box.
  2. Remova o cartao SD.
  3. Ligue novamente — o Armbian deve iniciar pela eMMC.
Verificacao rapida: lsblk deve mostrar / montado em mmcblk1p2 (eMMC) e nao em mmcblk0 (SD).

4Validar os servicos

Apos o boot pela eMMC, confirme que tudo esta funcionando:

# sistema
uname -r                                  # kernel 6.x
ip -br a                                  # IP na rede

# display frontal
systemctl status tx9-display.service
tx9-display-show show_text "OK" --duration 3

# controle remoto IR
systemctl status tx9-ir.service
tx9-ir-listen                             # aperte botoes no controle

# Wi-Fi
lsmod | grep ssv6051
nmcli device wifi list

4Usando Docker no TX9

O Docker ja vem instalado. Teste e comece a usar:

# verificar se esta rodando
systemctl status docker

# teste rapido
docker run --rm hello-world

# subir um servidor web
docker run -d --name web -p 80:80 nginx

# acessar no navegador do notebook:
# http://IP_DA_TVBOX
Resultado: voce tem um mini-servidor Linux com kernel moderno, rodando Docker, com display frontal e controle remoto IR funcionais — tudo a partir de uma TV Box que era lixo eletronico com Android inseguro.
Teste o display: tx9-display-show show_text "DOCK" --duration 3
